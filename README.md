# KUP — AI 인스타 운영 도구

> **만드는 시간은 줄이고, 발행 버튼은 내가 누른다.**
> 인스타를 갓 시작한 1인 인플루언서가 주제·컨셉에 맞춰 **카드뉴스를 기획·제작·발행·관리**하도록 돕는 AI 코파일럿.
> 더 빨리 만들고(가속) · 한곳에서 통합하고(통합) · 다음 액션으로 잇는(운영) — 검수·발행 주도권은 항상 사용자.

회원 기반 · 한국어 · 카드뉴스 중심 MVP. 최종 PRD + IA를 반영했습니다.

---

## 빠른 시작

```bash
npm install
cp .env.local.example .env.local   # (선택) Claude API 키 입력
npm run dev                        # http://localhost:3000
```

- **API 키 없이도 동작합니다.** `ANTHROPIC_API_KEY` 가 없으면 설문 기반 **템플릿 생성**으로 자동 대체.
- 키를 넣으면 전략·기획·카드 본문을 **Claude(`claude-opus-4-8`)** 가 생성(약 15초/건). 모델은 `ANTHROPIC_MODEL` 로 변경 가능.

## IA 구조

- **랜딩**: GNB(주요 기능·요금제·문의) + 히어로 + 기능/원칙 + FNB(약관·개인정보·고객센터)
- **인증**: 이메일 로그인 · 구글 연동 · 비회원 이용 / 회원가입(약관 동의·닉네임·이메일) / 온보딩 설문 + 환영 캐러셀
- **워크스페이스**: 상단 Header(계정 연동하기 · 요금제 · 프로필 모달) + 좌측 Sidebar
  - **연동 인스타 계정** — 다중 계정 연동·전환 (`/app/accounts`)
  - **AI 기획 리스트** — 전략 + 기획 백로그 테이블 + 기획 추가 + 제작하러가기 (`/app/plans`)
  - **콘텐츠 관리** — 6단계 칸반 / 리스트 토글 (`/app/board`)
  - **콘텐츠 성과** — 인스타 인사이트 · 업로드 기여 그래프 · 팔로워 챌린지 (`/app/insights`)
  - **DM 리드마그넷** — 댓글 트리거 → 정보 DM, 플랜 한도 (`/app/dm`)
- **마이페이지**: 계정 정보·비밀번호·마케팅 동의·콘텐츠 설정(설문)·구독/결제·데이터 관리(기간/전체 삭제) (`/app/mypage`)
- **요금제/결제**: 베이직/프로/프리미엄, 월/연 토글(연 30% 할인), 결제 스텁 (`/app/pricing`, `/pricing`)

### 6단계 콘텐츠 수명주기 (= 칸반 컬럼)

`기획중 → 기획완료 → 제작중 → 제작완료 → 예약업로드 → 업로드완료`

- **기획**: 주제·형식(카드뉴스/사진첨부형)·장수로 가벼운 아웃라인 생성
- **제작하러가기**: 아웃라인 기반 본문·캡션·해시태그·CTA 완성 → 에디터
- **검수 게이트**: ‘AI 생성물’ 표기(편집 시 해제) + 출처확인·민감표현·표기누락 → **사용자 승인** → 제작완료
- **발행**: 이미지(PNG) 생성 → 지금 발행 / 예약 발행 (내가 누르는 발행)

## 제품 원칙 (가드레일) — 코드에 반영

- **주도권**: 모든 산출물은 편집 가능한 초안. 최종 결정·발행은 사용자. (`src/lib/compliance.ts`)
- **검수 = 필수 게이트**: 미해결 플래그가 있으면 발행 버튼 비활성. 출처 확인은 휴먼인더루프 필수 체크.
- **민감 도메인 룰셋**: 금융·의료 등 권유/단정/보장 표현을 생성·검수 양쪽에서 차단.
- **AI 라벨**: ‘AI 생성물’ 표기, 사용자가 편집하면 자동 해제.
- **공식 API만**: 자동팔로우·콜드 DM·가짜 인게이지먼트 없음. DM은 옵트인 전제 + 플랜 한도(100/1000/무제한).
- **무인 자동발행 없음(P2)**: 예약은 사용자가 거는 예약.

## 기술 스택 · 스텁

- **Next.js 15 · React 19 · TypeScript · Tailwind v4**, 의존성 없는 JSON 파일 저장소(`.data/`), `scrypt` 세션.
- 생성 엔진 `src/lib/ai.ts` (Claude + 템플릿 폴백), 인스타 커넥터 `src/lib/ig.ts` (테스터 베타 시뮬레이터).
- **명확히 스텁(정식 출시 시 실연동)**: 구글 OAuth · PG 결제(포트원/토스) · 이메일 인증 · Meta 실발행/인사이트.
  모두 인스타 발행과 동일하게 시뮬레이터로 흐름을 검증합니다.

> 배포 전략: 지정 사용자 한정 베타 = Meta 개발 모드 + 테스터 계정. 앱 리뷰·사업자등록·정식 결제는 정식 출시 시 진행.

## 팀에 공유하기 / 배포

> ⚠️ 데이터는 로컬 파일(`.data/`)에 저장돼요. **각자 로컬로 돌리면 데이터는 각자 따로**, 팀이 **하나의 공유 인스턴스**를 쓰려면 한 곳에 배포하세요. 비밀키(`.env.local`)는 **절대 깃에 올리지 마세요**(이미 `.gitignore`로 보호됨). 키는 슬랙/1password 등 안전한 채널로 공유.

### 방법 1. 코드 공유 (Git) — 각자 로컬 실행 (개발용 권장)
```bash
# (최초 1회) 비공개 저장소 만들고 올리기
gh repo create kup --private --source=. --remote=origin --push
# 또는 수동:
#   git remote add origin git@github.com:<org>/kup.git && git push -u origin main
```
팀원은:
```bash
git clone <repo-url> && cd kup
npm install
cp .env.local.example .env.local      # 그리고 본인 ANTHROPIC_API_KEY 입력
npm run dev                            # http://localhost:3000
```

### 방법 2. Docker — 한 줄 실행 (환경 동일, 데이터 영구)
```bash
cp .env.local.example .env.local       # 키 채우기
docker compose up -d                   # http://localhost:3000  (데이터는 ./.data 에 저장)
docker compose logs -f                 # 로그
docker compose down                    # 종료
```

### 방법 3. 공유 인스턴스 배포 (팀이 같은 데이터로 사용)
**파일 저장소라 Vercel(서버리스)에는 안 맞아요.** 영구 디스크가 있는 Node 호스트를 쓰세요.

- **Render / Railway / Fly.io / VPS** 등에 위 Docker 이미지로 배포 + **영구 볼륨을 `/app/.data` 에 마운트**.
- 환경변수: `ANTHROPIC_API_KEY`, `SESSION_SECRET`(무작위 문자열), 그리고 인스타 발행 쓰면 `PUBLIC_BASE_URL`(= 배포된 공개 URL, ngrok 불필요), `IG_GRAPH_VERSION`.
- 배포 URL이 곧 공개 주소라, 인스타 발행도 ngrok 없이 바로 됩니다.

### 방법 4. 잠깐 라이브로 보여주기 (ngrok)
내 PC에서 `npm run dev` 켜둔 채:
```bash
ngrok http 3000        # → https://xxxx.ngrok-free.app  (이 주소를 팀에 공유)
```
> 내 PC가 켜져 있는 동안만 접속 가능. PUBLIC_BASE_URL 도 이 주소로 맞추면 인스타 발행까지 시연 가능.

### 무엇을 고를까
| 상황 | 추천 |
|---|---|
| 팀원이 각자 코드 보고 개발 | **방법 1 (Git)** |
| "그냥 켜서 써보고 싶다" | **방법 2 (Docker)** |
| 팀이 같은 데이터로 함께 운영 | **방법 3 (배포)** |
| 지금 잠깐 화면 공유 | **방법 4 (ngrok)** |

> 완전 서버리스(Vercel)로 가려면 JSON 파일 저장소를 DB(예: Postgres) + 객체스토리지(이미지/영상)로 바꾸면 돼요. 필요하면 도와드릴게요.
