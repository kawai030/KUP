## ① 포지셔닝 (한 줄 정의)

**기존(SPEC/요약본):**
> 주제·타깃만 고르면 AI 자동화가 카드뉴스·릴스를 매일 제작·발행하고 코멘트→DM으로 팔로워를 모아 첫 1,000명까지 도달한다.

**고도화:**
> 인스타 계정을 막 키우는 개인 크리에이터와 1인 브랜드가 **자신만의 톤을 유지하며** 꾸준히 콘텐츠를 만들 수 있도록, **주제 추천 · 콘텐츠 초안 · 직접 수정 · 성과 기반 다음 액션**을 제공하는 **AI 콘텐츠 성장 코치**. — *대신 운영이 아니라, 당신답게 만드는 시간을 줄인다.*

**핵심 메시지 전환**
- "대신 운영해드립니다" → **"당신답게 만들도록 시간을 줄여드립니다"**
- "1,000명 보장" → **"1,000명까지 가는 실행 루틴을 함께 설계합니다"**
- "자동 생성·자동 발행" → **"수정 가능한 초안과 다음 액션을 제안합니다"**

## ② 타깃 · JTBD

> SPEC의 'A·B·C = 3 생성 모드'를 인터뷰 실측으로 재정렬. 모드는 §7-콘텐츠 엔진으로 이동하고, 여기선 **결제·수용도 기준**으로 다시 나눈다.

### 1차 타깃 — 콜드스타트 초보 운영자 (인터뷰 A형)
> "계정을 키우고 싶고 할 일도 대충 아는데, 콘텐츠 만드는 시간이 너무 길고 반응이 없어 꾸준히 못 올린다. 내 문체·스타일은 유지하면서, 빠르게 초안 만들고 직접 고쳐서 올리고 싶다."
- 팔로워 0~1,000, 주제는 대략 정해짐, 릴스/카드뉴스/스토리 올릴 의지 있음
- GPT·Canva·CapCut 등 외부 툴 경험 있음 / 완전 자동화보다 **직접 수정 가능한 보조**를 선호
- 100·500명 같은 중간 목표에 동기 반응

### 2차 타깃 — 1인 브랜드·로컬 소상공인 (인터뷰 C형)
> "인스타가 실제 매출·문의 채널이다. 최종 비주얼은 내가 직접 보지만, 캡션·해시태그·릴스 오디오·시즌 기획·업로드 준비 같은 반복 보조는 맡기고 싶다. 브랜드 톤만 해치지 않으면 돈 낼 수 있다."
- 결제 가능성이 1차보다 **더 명확**(시간 절감 = 인건비 절감). 단 가격 민감(월 1~2만 원대)

### 결제 타깃 아님 / 벤치마크 활용 — 성장 경험자 (인터뷰 B형)
- AI 위임에 부정적. **초기 결제 타깃에서 제외.** 대신 "성공 운영자의 기준·노하우"를 **콘텐츠 추천·후킹 로직의 벤치마크**로 활용 (B 회고 데이터는 §7-2, 부록 참조)

### 초기 제외/후순위
- 이미 팔로워 많고 자기 브랜딩 기준이 강한 운영자 · AI가 맥락을 이해 못 한다고 보는 사용자 · 콘텐츠 제작을 순수 자기표현으로 보는 사용자 · 성장보다 기록 자체를 중시하는 사용자

## ③ 핵심 문제 (Pain) — 인터뷰 기반 5대

1. **제작 시간이 길어 업로드 빈도를 못 지킨다.** 촬영→대본→편집→캡션→해시태그→업로드까지 툴·기기 전환이 많다. (릴스 1편 ≈ 3시간)
2. **성과를 보지만 다음 액션으로 못 잇는다.** 조회·저장·공유·시청시간·프로필 방문을 *확인은* 하나, "그래서 다음에 뭘 바꾸지"가 체계화 안 됨.
3. **AI를 쓰지만 그대로 신뢰하지 않는다.** 최종 결과는 직접 검토·문체 수정. "자동 생성"보다 "수정 가능한 초안"을 원함.
4. **레퍼런스 탐색은 많지만 비체계적이다.** 돋보기 탭·캡처 저장 반복. 알고리즘이 관심사 안만 보여줘 사각지대 발생.
5. **톤앤매너·주체성 상실이 가장 큰 두려움이다.** 계정 = "나를 드러내는 공간/포트폴리오/명함". 획일적 AI 말투·조잡한 이미지·승인 없는 자동 발행이 신뢰를 깬다.

### Pain ↔ Reliever
| Pain | Reliever (이 제품) |
|---|---|
| 제작 시간·업로드 빈도 | 주제 추천 + 후킹·대본·카드뉴스 구성·캡션·해시태그·CTA **초안 패키지** |
| 성과→액션 단절 | 성과 기록 + **"배울 점 / 다음에 바꿀 점" 추천** |
| AI 불신 | **초안 + 직접 수정 에디터 + 최종 컨펌**(자동 발행 아님) |
| 레퍼런스 비체계 | **목적별 레퍼런스 아카이브 → 내 계정 맞춤 변환** (P1) |
| 주체성 상실 두려움 | **톤앤매너 프로필 · 금지 스타일 · 최종 컨펌 원칙** |

## ④ 해결 방향 · 제품 원칙

**해결 방향(한 문단):** 사용자의 계정 컨셉·톤앤매너를 기반으로 **주제와 초안을 제안**하고, 사용자가 **직접 수정한 뒤** 발행하도록 돕는다. 발행 후 **성과를 기록**하고 **다음 콘텐츠에서 바꿀 점을 추천**해 지치지 않는 실행 루틴을 만든다.

**제품 원칙 (의사결정 가드레일)**
1. **주체성 보존 우선.** 모든 산출물은 *수정 가능한 초안*. 계정의 톤·문체·기획 의도는 사용자 것이다.
2. **검수 = 직접 수정 + 최종 컨펌.** (SPEC의 "결정이지 작업 아님"을 인터뷰 근거로 수정) 자연어 지시·변형안 선택은 *보조*, 직접 편집이 *기본*.
3. **자동 발행은 신뢰 이후의 옵트인.** MVP 기본은 복사/내보내기. 자동 흐름은 신뢰 누적·명시 동의·킬 스위치 하에서만(M1+).
4. **공식 API만.** 자동팔로우·콜드 DM·가짜 인게이지먼트 금지(계정 정지·신뢰 리스크). 성장은 진짜 팬 기반으로만.
5. **AI 라벨·투명성 기본.** AI 생성 고지 + 승인 로그(감사 추적).
6. **작은 목표로 쪼갠다.** 1K는 보장이 아니라 100→500→1,000 마일스톤의 끝점.

## ⑤ MVP 범위 (P0) — 무엇을 먼저 만드나

> **MVP 한 줄:** 인스타 초보 운영자와 1인 브랜드가 자기 톤을 유지하며 꾸준히 콘텐츠를 만들도록, AI가 **주제 추천·초안·직접 수정·성과 기반 다음 액션**을 제공하는 코치. **자동 발행·완전 자동 운영·보장형 메시지는 제외.**

P0 7대 기능 (상세는 §7, 수용 기준은 FR 표):
1. **계정 온보딩 진단** — 상태 입력 → 현재 운영 단계 + 이번 달/주 목표
2. **계정 톤앤매너 프로필** — 키워드·문체 예시·금지 표현·캡션/해시태그/CTA/비주얼 가이드 저장(이후 모든 생성에 상속)
3. **콘텐츠 주제 추천** — 이번 주 5개 + 목적/포맷/후킹 방향/필요 소스
4. **콘텐츠 초안 생성** — 후킹·대본(30/60s)·자막·카드뉴스 구성·썸네일 문구·캡션·해시태그·CTA 패키지
5. **직접 수정 에디터** — 텍스트 직접 수정·문장별 대체안·톤 변경 버튼·해시태그 선택·발행 전 체크리스트·복사/내보내기
6. **성과 기록 + 다음 액션 추천** — (초기엔 수동 입력 허용) 지표 입력 → 강점/약점/다음에 바꿀 점/다음 실험
7. **초보자 계정 운영 체크리스트** — 프로페셔널 전환·프로필·링크·하이라이트·고정·인사이트 확인 등

## ⑥ 단계적 로드맵 (충돌 해소의 핵심)

> SPEC이 'MVP 주 기능'으로 둔 자동 발행·코멘트→DM·Graph API 깊은 연동을, 인터뷰 근거로 **신뢰가 쌓인 뒤 단계**로 재배치한다. (SPEC 기능을 버리는 게 아니라 *순서*를 바꾸는 것.)

| 단계 | 목표 | 핵심 기능 | API/자동화 수준 |
|---|---|---|---|
| **P0 (MVP)** | 제작 시간 단축 + 주체성 보존 신뢰 확보 | 진단·톤프로필·주제추천·초안·**직접수정**·성과기록·다음액션·체크리스트 | **인스타 연동 없이도 동작** — 복사/내보내기, 성과 수동 입력 가능 |
| **P1** | 리텐션·반복 사용 | 레퍼런스 아카이브(목적별·내 계정 변환) · 시즌 캘린더 · 트렌드 오디오/해시태그 · 콘텐츠 비중 관리 · DM/댓글 응대 템플릿 · 성장 사례/샘플 | IG **읽기 연동**(성과 자동 수집, Graph API insights) 도입 검토 |
| **P2 (신중)** | 운영 자동화(옵트인) | **승인 후 예약/자동 발행** · **코멘트→DM 리드마그넷** · 오토파일럿(저위험·신뢰 누적·킬스위치) | IG **쓰기 연동**(발행·메시지) + Meta 앱 심사 완료 필수 |
| **M2** | 확장 | 숏츠 완전 생성(증강 모드) · 멀티 계정 · 글로벌 | — |

> **배포 전략 = 지정 사용자 한정 베타 (앱 리뷰·사업자등록 안 함).** 실서비스 공개 출시 대신, Meta **개발 모드 + 테스터 계정**으로 지정된 베타 사용자만 연동·운영하고 리뷰를 모은다.
> - **근거:** 개발 모드에서는 앱에 역할(관리자·개발자·테스터)이 부여된 계정에 한해 발행·인사이트·메시지 API를 **앱 리뷰·비즈니스 인증 없이** 사용할 수 있다. "남의 계정에 연동"하는 부담(사업자등록·2~4주 심사·화면녹화)을 베타 단계에선 지지 않는다.
> - **부수 효과:** 원래 "앱 심사 후(P2)"로 미뤘던 **자동 발행·코멘트→DM도 베타 테스터 한정으로는 지금 검증 가능**하다. → 이 기능들을 뒤로 미루는 이유는 이제 *앱 심사 비용*이 아니라 **사용자 신뢰·UX**(C1) 하나로 좁혀진다.
> - **전제·한도:** 테스터는 인스타에서 **테스터 초대를 직접 수락**해야 함 · 비즈니스 계정 + FB 페이지 연결 필요 · 앱 역할 인원/호출 한도 존재(베타 규모엔 충분). 정식 공개 출시 시점에 가서 앱 리뷰·사업자등록을 재검토.

- **인증/데이터:** OAuth 연결, 비밀번호 미보관, 토큰 암호화. (연동 시점은 단계별)

## ⑩ KPI / 성공 지표 (단계별 적용)

> SPEC의 North Star("주간 팔로워 순증")는 *쓰기 연동·발행 규모*를 전제한다. MVP는 발행을 자동화하지 않으므로, **단계별로 운영 지표를 분리**한다.

**P0 운영 지표 (활성화·습관 — 인터뷰 §11 기반)**
- 활성화: 온보딩 완료율 · 첫 주제 선택률 · 첫 초안 생성률 · **초안 수정 완료율** · 복사/내보내기 완료율 · 첫 성과 기록률
- 습관: 주간 초안 생성 수 · 주간 수정 완료 수 · 성과 기록 재방문율 · 다음 액션 반영률 · **2주 연속 생성 사용자 비율**
- 가치 체감: 제작 시간 감소 체감 · 업로드 빈도 증가 · **"내 계정답다"고 평가한 콘텐츠 비율** · 100/500/1,000 마일스톤 달성률

**P1+ 운영 North Star(쓰기/읽기 연동 후):** 주간 활성 채널당 **팔로워 순증** `≈ 발행 수 × 게시물당 도달 × (도달→팔로우 전환율)`. 마일스톤(가설): 1개월 +100 / 3개월 누적 500 / 6개월 1,000.

**미션 지표(장기):** 100/500/1,000 마일스톤 도달 계정 수. *단일 보장이 아니라 단계로.*

**비즈니스:** 무료→유료 전환율 · 월간 유지율 · 유료 사용자 월 평균 콘텐츠 생성 수 · 소상공인 결제 전환율.

**가드레일(유지):** 게시물당 참여율(NSM 짝) · 무수정 승인율 60~85% · 평균 검수 시간 <1~2분 · 계정 제재·셰도우밴율 0 · 고위험 검수 누락률 0%.

## ⑫ 컴플라이언스 · 안전 · 리스크

**컴플라이언스(SPEC 계승):** AI 라벨 기본 · 승인 로그(감사 추적) · 도메인 규칙(금융=권유 표현 차단 / 패션·소상공인=협찬·뒷광고 자동 표기) · 고위험(미검증 사실·실존 인물·효능·규제 업종) 사람 검수 강제. EU AI Act §50 회색지대(오토파일럿)는 P2 전 법 자문 확정.

**리스크(인터뷰 §13)**
- *신뢰:* AI가 톤 이해 못 하면 신뢰 상실. 브랜드 무관 이미지·자극 문구·승인 없는 발행 특히 위험 → **최종 컨펌 원칙은 MVP 하드 룰**.
- *포지셔닝:* "자동화" 과도 강조 → 주체성 상실 우려 / "1K 보장" → 신뢰 하락 / "수익화"만 강조 → 취미·브랜딩 사용자 이탈.
- *기능 범위:* 처음부터 API 연동·자동 발행·트렌드 수집 다 넣으면 난이도 폭증 → **MVP는 수동 입력·복사/내보내기·체크리스트 중심**.
- *윤리:* 품앗이·자동 좋아요/팔로우 금지. "진짜 팬 + 콘텐츠 개선 기반 성장"을 메시지로.

## ⑱ Decision Log (왜 이렇게 정했나 — 재질문 방지)

- **자동 발행을 MVP에서 후순위로(C1):** 3명 모두 "그대로 올리기"에 신뢰 낮음 + 자동 발행이 신뢰 리스크. SPEC의 오토파일럿 "신뢰 누적 후" 철학과 모순 아님 — *기본 시작점*만 복사/내보내기로 이동.
- **검수를 직접 수정으로 강화(C2):** SPEC의 "검수=결정이지 작업 아님"은 인터뷰의 "본인 문체로 직접 수정" 공통 행동과 충돌. 자연어 지시만으론 부족 → 직접 편집을 기본, 자연어·변형안은 보조.
- **Aha 재정의(C3):** 가치는 자동화가 아니라 *시간 단축 × 주체성 보존*. "내가 안 했는데 올라갔다"는 일부 사용자에겐 불안.
- **타깃 재정렬(C4):** 결제·수용도 실측 반영. B는 결제 타깃 아님 → 벤치마크로 전환(성장 전술 데이터는 후킹·주제 로직에 활용).
- **1K 강등(C5):** 보장형 메시지 신뢰 리스크 + 초보 동기부여는 100/500 단계가 현실적.
- **North Star 단계 분리:** "팔로워 순증"은 쓰기 연동·발행 규모 전제 → P0는 활성화·습관·가치체감 지표로 운영, P1+에서 순증으로 승격.
- **유지: 3-레이어·공식 API만·AI 라벨·도메인 컴플라이언스·카드뉴스 메인** — SPEC 결정이 인터뷰와 충돌하지 않고 여전히 유효.
