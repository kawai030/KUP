**유지(v2.0에서 그대로 가져온 골격):** 3-레이어 분리 · 공식 API만 · 휴먼 인 더 루프 · 카드뉴스 메인 · 테스터 한정 베타(개발 모드) · 주체성 보존 원칙 · 단계적 API 연동.

### 🔎 충돌 처리 노트 (발행·DM이 MVP로 들어온 것에 대해)
v2.0은 **발행**과 **코멘트→DM**을 P2로 미뤘다. 팀 노트는 둘을 MVP에 넣었다. 모순이 아니라 **재정의로 해소**한다:
- **"자동 발행"(사람 확인 없이 게시) ≠ "검수·최종 컨펌 후 발행"**. MVP에 들어오는 것은 후자다. 최종 컨펌 원칙은 그대로 하드 룰.
- v2.0이 이미 적은 대로, **테스터 한정 베타(개발 모드)** 에서는 앱 심사 없이 발행·DM API를 쓸 수 있다 → 발행·DM을 미룰 이유가 "앱 심사 비용"에서 사라졌다. 남은 건 신뢰·UX였고, **검수 하드 게이트(§10)** 로 그 리스크를 관리한다.
- 따라서 **자동 발행(무인 게시)·오토파일럿은 여전히 P2 옵트인**, **검수 후 사용자 발행·옵트인 DM 리드마그넷은 MVP**.

## ① 포지셔닝 · 아하 모먼트 (T8)

**한 줄 정의**
> 인스타 계정을 막 키우는 개인 크리에이터와 1인 브랜드가, **자기 톤을 유지하며** 카드뉴스를 꾸준히 만들 수 있도록 — **사용자 정보 기반 전략 · 카드뉴스 초안 · 검수 · 편집·발행 · 성과 대시보드**를 제공하는 **AI 콘텐츠 성장 도구**. — *대신 운영해주는 게 아니라, "내가" 더 빨리 만들게 한다.*

**핵심 메시지 전환**
- "AI가 대신 운영" → **"AI가 시간을 줄이고, 최종 결정·발행은 내가 한다"**
- "자동 생성·자동 발행" → **"수정 가능한 초안 + 내가 누르는 발행"**
- "1,000명 보장" → **"100명씩 쌓아 1,000명까지 가는 실행 루틴"**

**아하 모먼트 (재정의 — 가장 중요)**
> **"AI 자동 생성으로 제작 시간이 확 줄었는데, 톤도 내 것이고 발행 버튼을 누른 것도 나다."**
- 가치의 핵심은 **자동화 자체가 아니라 ① 시간 단축 × ② 주체성(최종 주체=본인)**.
- 제품 전반에서 "자동화가 다 해줬다"가 아니라 **"내가 빠르게 해냈다"** 로 경험을 설계한다. (자동화는 수단, 주인공은 사용자)
- UX·카피·온보딩 모두 이 한 줄을 기준으로 검수: *자동화를 과시하지 말고, 사용자의 시간 절감과 통제감을 보여줄 것.*

## ② 타깃 · 제외 대상 (T1)

> v2.0의 결제·수용도 기준 재정렬을 유지하되, **제외군을 "지금은 안 받는다"로 구체화**한다.

### 1차 타깃 — 콜드스타트 초보 운영자
- 팔로워 **0~1,000**, 주제는 대략 정해짐, 카드뉴스/릴스/스토리를 올릴 의지는 있음
- 매번 만드는 **시간이 길고 반응이 없어** 꾸준히 못 올림
- GPT·Canva·CapCut 경험 있음 / 완전 자동화보다 **직접 수정 가능한 보조**를 선호
- **100·500명 같은 중간 목표에 동기 반응** → 단계 로드맵과 궁합

### 2차 타깃 — 1인 브랜드 · 로컬 소상공인
- 인스타가 **실제 매출·문의 채널**. 캡션·해시태그·시즌 기획·업로드 준비 같은 **반복 보조**를 위임하고 싶음
- 결제 가능성이 1차보다 명확(시간 절감=인건비 절감), 단 **가격 민감(월 1~2만 원대)**

### 결제 타깃 아님 / 벤치마크로만 활용 — 성장 경험자
- AI 위임에 부정적. **초기 결제 타깃에서 제외.** 대신 성공 운영자의 기준·후킹 노하우를 **추천·후킹 로직의 벤치마크 데이터**로만 사용.

### 초기 제외 대상 (지금은 받지 않는다 — 구체화)
| 제외군 | 왜 지금은 제외하나 | 재검토 시점 |
|---|---|---|
| **이미 팔로워 多·자기 기준 강한 운영자** | 톤 통제 욕구가 강해 AI 초안 가치가 낮음 / 1K 콜드스타트 문제와 무관 | M2(고급 편집·멀티계정) |
| **"AI는 내 맥락을 모른다"고 단정한 사용자** | 신뢰 전제가 깨져 검수·수정 가치 전달 불가 | 성공 사례 축적 후 |
| **콘텐츠를 순수 자기표현·기록으로만 보는 사용자** | 성장·수익화 동기가 없어 단계 로드맵·성과 루프가 작동 안 함 | 비대상(포지셔닝 불일치) |
| **민감 규제 업종 전문 영업 목적**(금융상품 판매·의료 시술 유치 등) | "권유/강요" 표현 제거 룰(§10)과 충돌, 법적 리스크 큼 | 도메인 컴플라이언스 정식화 이후 |
| **중장년·은퇴층** | 현 타깃 검증 범위 밖(인터뷰 미포함) | 1차 검증 후 별도 검토(§17) |
| **글로벌·다국어 사용자** | 한국 우선 MVP | M2 |

## ③ 페르소나 (T7 · 재작성)

> 재정렬된 1차/2차 타깃에 맞춰 페르소나를 다시 쓴다. (요약본의 '민준' 단일 페르소나 → 2종으로 분화)

### 페르소나 A — 지우(29) · 콜드스타트 초보 *(1차)*
- **상황:** 직장인. "퇴근 후 운동·식단" 주제로 6개월 전 계정 시작. 팔로워 280명에서 정체.
- **행동:** 카드뉴스 1개 만드는 데 2~3시간. 반응이 없어 2주에 한 번도 못 올림. GPT로 초안은 뽑지만 "내 말투가 아니라" 매번 다 뜯어고침.
- **목표:** 시간 안 들이고 **꾸준히** 올려서 일단 **1,000명**까지. 단, 내 톤·얼굴·관점은 유지.
- **두려움:** 획일적 AI 말투로 "남의 계정처럼" 보이는 것. 승인 없이 뭔가 올라가는 것.
- **이 제품에서:** 설문→전략→카드뉴스 초안→**내 톤으로 수정**→발행. 대시보드의 **100명 단위 로드맵**으로 "다음 100명"에 동기.

### 페르소나 B — 현수(41) · 1인 브랜드 소상공인 *(2차)*
- **상황:** 동네 베이커리 사장. 인스타가 사실상 유일한 마케팅 채널. 신메뉴·시즌 행사 알림이 곧 매출.
- **행동:** 사진은 직접 잘 찍지만 캡션·해시태그·업로드 타이밍이 늘 밀림. 비주얼 최종은 꼭 본인이 확인.
- **목표:** **반복 작업(캡션·해시태그·시즌 기획·업로드 준비)** 을 줄이고 문의·방문을 늘리기. 브랜드 톤만 안 해치면 **월 1~2만 원** 지불 의향.
- **두려움:** 협찬/뒷광고 표기 누락, 브랜드와 안 맞는 톤, 자동으로 잘못 올라가는 사고.
- **이 제품에서:** 보유 사진(자산)→카드뉴스 가공→검수(협찬·표기 자동 점검)→발행. **문의·방문 지표**로 다음 액션.

## ④ 사용자 입력 정보 (T2 · T15 · T14)

> 입력을 **2계층으로 명확히 분리**한다. (A) 시작 시 **사용자 설문**(사람·계정 맞춤화용, 콘텐츠와 별개) / (B) **콘텐츠별 입력**(카드뉴스 1건을 만들 때). 그리고 **내가 기획한 주제 직접 입력 모드(T14)** 를 정식 추가.

### A. 시작 설문 — "당신은 어떤 사람/계정인가" (1회, 이후 모든 생성에 상속)
> **카드뉴스 정보와는 별개.** 맞춤형 제작을 위한 사람·계정 프로필. 온보딩에서 한 번 받고 마이페이지에서 수정.

- **계정 기본:** 주제(니치) · 현재 팔로워 수 · 운영 기간 · 운영 목적(취미/브랜딩/협찬/매출/문의/포트폴리오)
- **운영 여건:** 주당 업로드 가능 횟수 · 주 콘텐츠 형식(카드뉴스/릴스/사진/스토리) · 보유 자산(사진·영상·제품/매장)
- **톤앤매너:** 브랜드 키워드 3~5 · 문체 예시 · **금지 표현/스타일** · 선호 캡션 길이 · 해시태그 스타일 · CTA 스타일 · 비주얼 가이드(색감·무드)
- **민감도:** 도메인이 금융·의료 등 규제 영역인지(→ §10 표현 금지 룰 자동 적용)
- **벤치마크:** 참고하고 싶은 계정(있으면)

→ 출력: **운영 단계 진단**(세팅/누적/반응 탐색/성장 실험/수익화 준비) + **이번 주 콘텐츠 실행 목표**(루틴, §9)

### B. 콘텐츠별 입력 — "이번 카드뉴스를 무엇으로" (매 제작)
- **주제 소스 2모드:**
  - **(B-1) 추천 주제 선택** — 시스템이 이번 주 N개 추천(목적·후킹 방향 포함) 중 고름
  - **(B-2) 내 주제 직접 입력 [T14]** — *사용자가 기획한 주제·메시지·소재를 직접 넣어 생성.* 추천에만 의존하지 않고 본인 기획을 그대로 살림 (예: "이번 주 신메뉴 출시 알림", "내가 정리한 3가지 운동 루틴")
- **콘텐츠 입력:** 목적(조회/저장/공유/방문/문의) · 포맷(카드뉴스 페이지 수) · 핵심 메시지/소구점 · 필요 소스(직접 보유 사진/문구) · 톤 미세조정(설문값 기본 + 이번 건만 변경)

> **설계 원칙:** A는 "사람을 안다", B는 "콘텐츠를 만든다". A 없이 B를 받으면 톤이 흔들리고, A만 받고 B를 안 받으면 일반론이 된다. **B-2(직접 입력)는 1급 시민** — 추천과 동등하게 취급한다.

## ⑤ 1,000명 목표 — 근거와 단계화 (T3 · T4 · T5)

### 5-1. 왜 "1,000명"인가 (T3 — 근거 프레임)
> 1,000을 **목표 슬로건이 아니라 "콜드스타트 통과선"** 으로 근거화한다. *아래는 근거 가설 — 일부 수치는 출처 확인/리서치 보강 필요(🔬 TODO).*

| 근거 축 | 내용 | 상태 |
|---|---|---|
| **콜드스타트 이탈 구간** | 0→1,000은 초기 무반응으로 **이탈이 가장 큰 구간**. 이 구간만 넘기면 추천 유입·자생이 시작 → 우리가 풀려는 문제와 정확히 일치 | 인터뷰로 정성 확인(synthesis) |
| **수익화 진입선** | 마이크로 인플루언서/소상공인 기준 브랜드 협찬·제휴 문의가 들어오기 시작하는 **최소 신뢰 규모**로 통용 | 🔬 출처 확인 필요 |
| **추천 도달 자기강화** | 일정 규모 이상에서 비팔로워 추천 도달률이 올라가는 구간 진입(저장·공유 신호 누적) | 🔬 플랫폼 데이터 확인 필요 |
| **'1,000 진성 팬' 담론** | 직접 수익화 가능한 최소 진성 팬 규모 개념(소상공인엔 문의·재방문으로 환산) | 개념적 근거 |

> **TODO(근거 보강):** 위 🔬 항목은 공개 데이터·리포트로 출처를 붙여야 한다. 필요 시 별도 리서치로 인용 자료를 수집해 이 표를 채운다. *현재는 "콜드스타트 통과선"이라는 정성 근거가 주력이며, 정량 출처는 보강 대상.*

### 5-2. 100명 단위 단계 로드맵 (T4 · T5 — 기간 제외)
> **바로 1,000을 걸지 않는다.** 100명 단위로 끊어 **"다음 100명"** 을 단기 동기 단위로 삼는다. **기간(개월) 목표는 일단 제외** — 수량 단계만 둔다.

```
[0]──100──200──300──400──500──600──700──800──900──[1,000]
 세팅   첫 반응    누적·실험        성장 가속        수익화 진입선
```
- **단계 = 동기 단위:** 각 100 구간 도달 시 대시보드에서 시각적 마일스톤(§11). "1,000은 멀어도 다음 100은 가깝다."
- **기간 미설정:** "1개월/3개월/6개월" 같은 기한 목표는 **이번 버전에서 제외**(개인별 여건 편차가 커 기한 약속이 신뢰 리스크). 추후 데이터가 쌓이면 *참고용 페이스*로만 재검토(§17).
- **보장 아님:** 1,000은 "보장"이 아니라 **단계 로드맵의 끝점**. 메시지는 "단계별 실행 루틴"으로.

## ⑥ 핵심 문제 (Pain) — 인터뷰 기반 (유지)

1. **제작 시간이 길어 업로드 빈도를 못 지킨다.** (카드뉴스 1건 = 2~3시간, 툴 전환 잦음)
2. **성과를 보지만 다음 액션으로 못 잇는다.** "그래서 뭘 바꾸지"가 체계화 안 됨.
3. **AI를 쓰지만 그대로 신뢰하지 않는다.** 최종은 직접 검토·문체 수정. "자동 생성"보다 "수정 가능한 초안".
4. **레퍼런스 탐색이 비체계적이다.** (→ P1 아카이브)
5. **톤앤매너·주체성 상실이 가장 큰 두려움.** 획일적 말투·승인 없는 자동 발행이 신뢰를 깬다.

| Pain | Reliever |
|---|---|
| 제작 시간·빈도 | 사용자 정보 기반 전략 + **카드뉴스 초안 패키지** |
| 성과→액션 단절 | 대시보드 성과 기록 + **다음에 바꿀 점 추천** |
| AI 불신 | **초안 + 직접 편집 + 검수 게이트 + 내가 누르는 발행** |
| 주체성 두려움 | **톤 프로필 · 금지 표현 · 최종 컨펌 원칙 · 내 주제 직접 입력(B-2)** |

## ⑦ 해결 방향 · 제품 원칙

**해결 방향:** 시작 설문으로 계정 컨셉·톤을 파악 → **전략과 카드뉴스 초안을 제안**(또는 내 주제로 생성) → 사용자가 **검수·직접 편집** 후 → **본인이 발행** → 대시보드에서 **성과·진척**을 보고 **다음 액션**으로 잇는, 지치지 않는 **주 2회+ 루틴**.

**제품 원칙 (가드레일)**
1. **주체성 보존 우선.** 모든 산출물은 *수정 가능한 초안*. 최종 결정·발행 주체는 사용자.
2. **검수 = 직접 수정 + 최종 컨펌 + AI 라벨.** 발행 전 검수는 **필수 게이트**(§10).
3. **자동 발행(무인 게시)은 P2 옵트인.** MVP의 발행은 *검수 통과 후 사용자가 누르는 발행*.
4. **공식 API만.** 자동팔로우·콜드 DM·가짜 인게이지먼트 금지.
5. **AI 라벨·투명성 기본.** 'AI 생성 콘텐츠' 표기 + 승인 로그.
6. **작은 목표로 쪼갠다.** 100→1,000 단계, 기간 약속 없음.
7. **꾸준함 우선(루틴).** 주 2회+ 발행 루틴을 제품이 떠받친다(§9).
8. **내 기획도 1급.** 추천뿐 아니라 *사용자가 기획한 콘텐츠도 직접 생성*(§4 B-2).

## ⑧ MVP 범위 (P0) — 팀 정의 7대 기능 (T9)

> **MVP 한 줄:** 사용자 정보 기반 **전략 → 카드뉴스 제작 → 검수 → 편집·발행 → 대시보드/마이페이지 → DM 리드마그넷**. **자동 발행(무인)·완전 자동 운영·보장형 메시지는 제외.** 릴스 스크립트·트렌드 아카이브는 다음 단계.

| # | 기능 | 핵심 내용 | 비고 |

| 1 | **사용자 입력 정보 기반 콘텐츠 제작 전략 제시** | 시작 설문(§4-A) 기반 운영 단계 진단 + 이번 주 전략/주제 방향 + 주제 추천 N개 | 추천 + **내 주제 직접 입력(B-2)** 둘 다 |

### 8-1. 보강 메모
- **③ 검수**와 **④ 편집·발행**은 한 흐름이지만 기능 책임이 다르다: 검수=**게이트(통과/반려)**, 편집=**수정**, 발행=**사용자 액션**. 셋 다 같은 화면에서 연결되되 검수 통과 없이는 발행 버튼이 활성화되지 않는다.
- **⑤ 대시보드의 100단위 로드맵**은 동기부여 UI이지 측정 지표가 아니다(T10). KPI는 §12.

## ⑨ 콘텐츠 생성 루틴화 (T6)

> "한 번 만들고 끝"이 아니라 **꾸준함**이 1,000명의 진짜 변수. 제품이 **주 2회 이상** 루틴을 떠받친다.

- **권장 기본값: 주 2회 이상.** 온보딩 설문의 "주당 업로드 가능 횟수"를 받아, **주 2회를 권장 하한**으로 제시(여건상 1회면 1회로 시작하되 2회 상향을 유도).
- **루틴 장치:**
  - 이번 주 **목표 발행 수** 표시 + 진행 바(예: 1/2)
  - 미달 시 **리마인드/주제 미리 제안**(다음 콘텐츠 초안을 미리 준비)
  - 대시보드에 **연속 발행 주차(streak)** 표시 → 끊기지 않게
- **지표 연결:** 루틴 준수(주 2회+) 사용자의 성과·잔존을 KPI로 추적(§12) → "꾸준함=성장" 가설 검증.
- **주의:** 루틴은 *압박이 아니라 도움*. 톤은 격려형, 미달을 벌하지 않음(이탈 방지).

## ⑩ 검수 · 컴플라이언스 (T12 · T13)

> **검수는 두 종류다 — 섞지 않는다 (결정 2026-06-25):**
> ① **품질 검수**(톤·후킹·구조·문장 완성도) = **생성 단계에 흡수**, 사용자에겐 별도 화면/대기로 안 보임. 초보가 받는 첫 초안(K2)이 발행 가능 수준이 되도록 self-critique 1패스. → *편집 전, 비가시.*
> ② **안전 검수**(미검증 주장·효능·실존인물·표기 누락·AI라벨) = **발행 직전 하드 게이트(K4)로 노출.** 사용자가 편집 중 새로 친 사고(틀린 가격·효능 단정 등)를 잡아야 하므로 **반드시 편집 *뒤*** — 편집이 사전 검수를 무효화하기 때문.
> **우선순위:** 안전 게이트 = **day-1 필수** / 품질 패스 = 생성 토큰 ~2배라 **측정 후 fast-follow**(우선은 생성 프롬프트 보강으로 흡수, 효과 측정 후 별도 refine 패스 가동).

### 10-1. 'AI 생성 콘텐츠' 명시 + 검수 절차 = 필수 게이트 (T12)
> 검수를 **선택이 아니라 발행 전 통과 필수 단계**로 승격한다.

- **AI 생성 표기 필수:** 모든 산출물에 **'AI 생성 콘텐츠'** 라벨/고지를 기본 부착(사용자가 끌 수 없는 표기 정책은 정식 출시 전 확정).
- **검수 게이트:** 발행/내보내기 전 **검수 화면을 반드시 경유.** 시스템이 자동 플래그한 항목(민감 표현·표기 누락·미검증 주장·이미지 아티팩트)을 사용자가 확인/수정해야 **발행 버튼 활성화.**
- **승인 로그:** 누가·언제 무엇을 승인했는지 감사 추적 기록.
- **빈 검수 방지:** 전부 다시 보라는 게 아니라 *플래그된 항목 중심*으로 판단(검수 시간 <1~2분 목표).

### 10-2. 민감 도메인 표현 금지 룰셋 (T13)
> 금융·의료 등 규제·민감 영역은 **'권유'·'강요'·단정·보장 표현을 철저히 제거.**

| 도메인 | 금지/주의 표현 (예) | 대체 방향 |
|---|---|---|
| **금융·투자·부동산** | "지금 사라/들어가라", "무조건 오른다", "보장 수익", "추천 종목" 등 **권유·단정** | 정보 제공·일반 교육형("일반적으로 ~한 구조다"), 면책 고지 |
| **의료·건강·다이어트** | "이거 먹으면 낫는다", "반드시 효과", "병원 안 가도", **효능 단정·치료 권유** | 일반 정보·"개인차 있음·전문가 상담 권장" 고지 |
| **기타 규제(법률·세무 등)** | 단정적 조언·"이렇게 하면 무조건" | 일반 정보 + 전문가 확인 안내 |

- **적용 방식:** 설문(§4-A)에서 도메인이 민감군으로 표시되면 **해당 룰셋을 생성·검수 양쪽에 자동 적용.** 검수 단계에서 금지 표현 패턴을 플래그하고, 생성 단계에서 애초에 권유형 문장을 회피하도록 제약.

## ⑫ KPI / 성공 지표 (T10 · T11 · T5)

> **변경 핵심:** ① 100명 단위는 **KPI에서 제외**(대시보드 노출용) ② **시간 단축·편의 만족도**를 핵심 KPI로 신설 ③ **기간 목표 제거**.

### 12-1. 핵심 KPI — 가치 증명
- **🌟 제작 시간 단축·편의 만족도 (신설, T11):** *전체 이용자 중* 콘텐츠 제작 **시간이 줄었다 / 편해졌다**고 응답한 비율(만족도/주관 체감). 보조: 실측 제작 소요시간 감소율(생성→발행까지). → **이 제품의 1차 가치(시간 단축×주체성)를 직접 측정.**
- **활성화:** 온보딩 설문 완료율 · 첫 전략 수신율 · 첫 카드뉴스 생성률 · **검수·편집 완료율** · 발행/내보내기 완료율
- **습관(루틴):** **주 2회+ 발행 루틴 준수율** · 주간 생성 수 · 연속 발행 주차 · 2주 연속 생성 사용자 비율 · 다음 액션 반영률
- **비즈니스:** 무료→유료 전환율 · 월 유지율 · 유료 사용자 월평균 생성 수 · 소상공인 결제 전환율

### 12-2. KPI에서 **제외**한 것
- ❌ **팔로워 100명 단위 달성률·마일스톤 달성률** — *사용자에게 보여주는 동기부여 지표(대시보드 §11)일 뿐, 서비스 성과 KPI로는 부적합*(외부 변수 많고 우리 통제 밖). KPI 아님.
- ❌ **기간 기반 목표**(1개월/3개월/6개월 +N) — 이번 버전 제외(§5-2).

### 12-3. P1+ (읽기/쓰기 연동 후)
- 운영 North Star로 **주간 활성 채널당 팔로워 순증** `≈ 발행 수 × 게시물당 도달 × (도달→팔로우 전환율)` 승격(기간 약속 없이 추세로).

### 12-4. 가드레일 (유지)
게시물당 참여율(NSM 짝) · 무수정 승인율 60~85% · 평균 검수 시간 <1~2분 · 계정 제재·셰도우밴율 0 · **고위험/민감표현 검수 누락률 0%** · DM 리드마그넷 스팸 신고율(임계 감시).

## ⑱ Decision Log (왜 이렇게 정했나 — 재질문 방지)

- **발행·DM을 MVP로 (T9):** "자동 발행(무인)"이 아니라 "검수 후 사용자 발행"이며, 테스터 베타가 앱 심사 장벽을 없애 지금 검증 가능 → v2.0의 P2 배치를 재정의해 MVP로 당김. 최종 컨펌·옵트인·공식 API 원칙은 유지(§0).
- **릴스·트렌드 아카이브를 다음 단계로 (T9):** MVP는 카드뉴스에 집중해 범위·난이도 축소. 릴스 스크립트·아카이브는 P1.
- **아하 재정의 (T8):** 가치는 자동화가 아니라 *시간 단축 × 주체성*. "최종 주체는 본인"을 메시지·UX 기준선으로.
- **1,000 근거화 + 100단위 단계 + 기간 제외 (T3·T4·T5):** 1,000은 "콜드스타트 통과선"으로 근거화하되 보장 아님. 동기는 '다음 100명' 단위. 기한 약속은 신뢰 리스크라 제거.
- **100단위는 KPI 아님 / 만족도는 KPI (T10·T11):** 팔로워 단계는 외부 변수가 커 측정 지표로 부적합 → 대시보드 동기 UI로. 우리가 통제·증명할 가치는 "시간 단축·편의 만족도" → 핵심 KPI로 신설.
- **검수 필수 게이트 + 민감 표현 제거 (T12·T13):** AI 라벨·검수를 발행 전 하드 게이트로 승격. 금융·의료 등은 권유/단정/보장 표현을 생성·검수 양단에서 차단.
- **검수 2종 분리 — 품질(편집 전·비가시) / 안전(편집 후·게이트) (2026-06-25):** "AI가 한 번 검수 후 사용자 편집" 요구를 두 검수로 분해. 품질은 생성에 흡수해 초보가 깨끗한 초안을 받게 하고, 안전은 편집 *뒤* 하드 게이트로 둠(편집이 사전 검수를 무효화하므로). 안전 게이트 day-1 필수, 품질 패스는 토큰 ~2배라 측정 후 가동. (§10 머리말)
- **렌더 엔진 유지 + 편집 범위 제한 (2026-06-25):** 편집 가능한 미리보기를 넣되 렌더 엔진은 서버 Playwright(HTML→PNG) 그대로 — 미리보기=브라우저, 확정=서버로 폰트/줄바꿈 일관성 확보. 편집은 텍스트·사진·스타일 3종 한정(자유 캔버스화 방지 — 'AI가 해준다' 가치와 충돌). (§14)
- **입력 2계층 분리 + 내 주제 직접 입력 (T2·T15·T14):** 사람(설문)과 콘텐츠(제작)를 분리해 톤 일관성 확보. 추천에만 의존하지 않도록 사용자 기획 콘텐츠를 1급 기능으로.
- **페르소나 재작성 (T7):** 재정렬된 1차/2차 타깃에 맞춰 '지우/현수' 2종으로.
