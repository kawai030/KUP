## 🌿 git worktree 운영 원칙

- **브랜치 성격**: `spike/*`(버림) vs `feat/*`·`dev`(머지). Task 3+는 `dev`(머지 대상).
- **add는 머신당 1회.** 평소엔 창 2개만 열면 됨. worktree 폴더는 머신 로컬(git 동기화 X) — 다른 노트북은 거기서 1회 add.
- **worktree마다 독립**: `node_modules`·`.env` 공유 안 됨 → 각자 `npm install`+자체 `.env.local`. 시크릿 커밋 금지(`.env.*` gitignore됨).
- **머지 케이던스**: 각 Task 검증 기준 달성 시 dev→main → 기획 문서와 코드 수렴.
- **머신 간**: `dev` 브랜치 push/pull로 동기화.
