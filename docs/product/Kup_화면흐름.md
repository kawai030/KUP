# Kup — 화면 흐름 (유저 저니)

> **문서 성격**: [IA](Kup_IA.md)의 화면 계층을 실제 **클릭 동선**으로 푼 문서. 와이어프레임·구현의 화면 단위·전이·분기 기준.
> **기준일**: 2026-06-26 · **선행**: [IA 최종본](Kup_IA.md) · [SPEC §3](Kup_SPEC.md) · [데이터 모델](../tech/Kup_데이터모델.md)
> **모델**: 미리캔버스형 — 공개 인트로 → 로그인/회원가입 **모달** → **워크스페이스**(로그인 후 모든 작업 단일 셸).
> ⚠️ **역할 분담(와이어프레임 병행)**: 완성본 **와이어프레임 프로토타입 = 화면 레이아웃·인터랙션의 시각 기준**. **본 문서 = 화면 전이·게이트·데이터 바인딩의 로직 기준**(상태 전이·하드게이트·테이블 매핑 등 와이어프레임이 담지 않는 것). 둘은 경쟁이 아니라 **보완**이며, **시각·동선이 충돌하면 와이어프레임을 우선**해 본 문서를 갱신한다(상세 §7).

## 1. 전체 내비게이션 맵

```
[L0 랜딩] ──[시작하기]──> (L1 로그인 / L2 회원가입 모달)
                                   │
                        ┌──────────┴───────────┐
                   회원가입 완료            비회원으로 이용
                        │                       │
                        ▼                       ▼
              [O1 설문]→[O2 컨셉]→[O3 연동]   [W0 비회원 셸(제한)]
                        │                       │ (저장·발행 시도 → L2 가입 유도)
                        ▼                       
                  ┌─────────────── [W0 워크스페이스 셸] ───────────────┐
   좌측 사이드바  │  ① 연동 인스타 계정   ② 콘텐츠 관리   ③ 자동화 설정  │
                  │  ④ 콘텐츠 성과       ⑤ 프로필                     │
                  └───────────────────────────────────────────────────┘
                        │
   ① ─> [C1 기획리스트]→[C2 초안생성]→[C3 편집]→[C4 검수게이트]→발행/예약
   ② ─> [M1 칸반]⇄[M2 캘린더/리스트]
   ③ ─> [G1 리드마그넷 설정]
   ④ ─> [P1 인사이트] · [P2 챌린지] · [P3 릴레이]
   ⑤ ─> [A1 프로필 모달]→[A2 마이페이지]→[A3 요금제·결제]
```

> 발행 기능은 어디서 진입하든 **C4 검수 게이트를 반드시 통과.** 미연동 상태면 발행·연동 의존 기능 클릭 시 `GATE-연동` 모달이 먼저 뜬다.

## 2. 핵심 플로우 5개

### Flow ① 온보딩 → 컨셉 확정 *(SPEC 3.1)*

```
L2 가입완료 → O1 설문 → O2 컨셉 제시 → (확정) → O3 연동 → W0
```

1. **O1 설문** — 주제·톤·금지표현·발행 빈도 / 타깃과 타깃의 결핍 / 제공 가치 입력.
2. **O2 컨셉·추천주제** — AI가 **계정 컨셉**(페르소나·톤·기둥·비주얼)과 **추천 주제 목록** 생성·제시.
   - 분기: [다시 생성] → O1 일부 수정 후 재생성 / [이대로 확정]
3. **컨셉 확정(lock)** — 이후 모든 생성의 제약으로 상속. → `O3` 연동 유도.
4. **O3** 로 이동(Flow ②).

🔗 **데이터**: `channel_configs`(persona/tone/pillars/cadence/visual + survey_raw, `locked_at`). 설문 원본은 재생성·디버그 위해 `survey_raw` 보존.
🟦 **빈/예외**: 설문 미완료 시 O2 진입 불가(필수 필드 가드).

---

### Flow ② IG 계정 연동 *(SPEC 3.1 / 2.2)*

```
O3(또는 사이드바 ①/상단 [+계정 연동]) → 연동 가이드+안전문구 → OAuth → 성공/실패
```

1. **연동 안내** — 전제 조건 체크리스트(비즈니스/크리에이터 계정 전환·페이지 연결) + **안전 안심 문구**(비밀번호 미보관, OAuth·토큰 암호화) 노출.
2. **OAuth** — Meta 인증창 → 권한 동의.
3. 분기:
   - ✅ **성공** → `channels.status='connected'`, 토큰 암호화 저장 → W0 진입(또는 직전 작업 복귀).
   - ❌ **실패/전제 미충족** → 무엇이 필요한지 안내, 발행 기능은 **잠금 유지**(`needs_setup`).

🔗 **데이터**: `channels`(status), `ig_tokens`(`access_token_enc`🔒, expires_at).
🚧 **게이트 `GATE-연동`**: 미연동 상태에서 발행/인사이트/리드마그넷 등 연동 의존 기능 클릭 → "SNS 계정 연동하기" 모달 → O3로 유도.

---

### Flow ③ 제작: 기획 → 초안 → 편집 *(SPEC 3.2)*

```
C1 기획리스트 → [제작하러 가기] → C2 초안생성(AI) → C3 편집(자유)
```

1. **C1 AI 기획 리스트** — 테이블(No / 형식(카드뉴스·사진첨부형) / 장수 / 타이틀(주제) / 서브타이틀+본문×장수 / [제작하러 가기]). [+기획 추가] 가능.
   - 칸반 상태: 행 생성 `planning` → 주제·전략 확정 `planned`.
2. **C2 초안 생성** — [제작하러 가기] → AI가 컨셉+주제로 **deck JSON 생성 → 렌더 PNG**. 로딩 상태 표시.
   - 상태: `producing` → 완료 시 `produced`.
3. **C3 편집** — **항상 편집 가능.** 텍스트·사진·스타일(템플릿/브랜드컬러) 수정. **편집 중에는 검수를 반복 강제하지 않음**(상시 재검수 루프 폐기).

🔗 **데이터**: `decks`(slides/caption/hashtags/lead_keyword … = [생성 파이프라인 §1.2 계약](../tech/Kup_생성파이프라인_설계.md)과 1:1). 상태 전이 `planning→planned→producing→produced`.
🟦 **빈 상태**: 기획 0건 → "첫 기획 추가하기" 안내 카드.
🚧 **게이트**: 미연동이면 C1 진입/제작은 가능하되, **발행(C4) 시점에 `GATE-연동`**.

---

### Flow ④ 발행: 검수 게이트 *(SPEC 3.3 / 3.5 · 차별화 핵심)*

```
C3 → [발행] 또는 [예약] → C4 자동검수 1회 → 승인 → 발행/예약 → 칸반 published/scheduled
                                    └ 거절 → C3 편집 복귀
```

1. **[발행]/[예약] 클릭** → **C4 검수 게이트 자동 1회 실행**(별도 검수 버튼 없음).
2. **C4 검수 모달**:
   - **AI 자가점검 플래그** 표시 — 출처/민감표현(권유·강요)/표기 누락. **플래그된 항목만** 강조(없으면 "이상 없음").
   - **'AI 생성 콘텐츠' 라벨** 확인(편집 거치면 해제 규칙 적용).
   - **사용자 최종 승인/거절**(필수).
3. 분기:
   - ✅ **승인 + 즉시 발행** → Graph API 2단계(컨테이너→FINISHED→게시) → `posts`. 칸반 `published`.
   - ✅ **승인 + 예약** → `schedules` 등록(BullMQ delayed job) → 칸반 `scheduled` → 예약시각 발행.
   - ❌ **거절** → C3 편집 복귀. (수정 후 다시 발행 누르면 검수 **재실행**)
4. **발행 결과** — 성공(완료·permalink) / 실패(사유·재시도).

🔗 **데이터**: `reviews`(decision/ai_flags/approved_by/decided_at = 감사로그), `schedules`(bullmq_job_id⏱), `posts`(ig_media_id).
🚧 **하드 게이트**: 검수 미통과·미승인 시 **발행 불가**(MUST NOT). `produced→scheduled/published` 전이는 `reviews.decision='approved'` 전제.
🟥 **에러**: 발행 실패 → `schedules.status='failed'`+`last_error`, 재시도 CTA / 토큰 만료 시 재연동 유도.

### Flow ⑤ 대시보드: 성과 → 다음 액션 *(SPEC 3.7)*

```
사이드바 ④ → P1 인사이트 / P2 챌린지 / P3 릴레이 → 다음 액션 제안
```

1. **P1 인스타 인사이트** — 계정 단위(팔로워 수·유입/이탈) + 게시물 단위(조회·도달·저장·공유·좋아요·댓글·프로필방문·기여팔로우) + 리드마그넷 자체 카운트.
   - **갱신 시각 표기**(`YYYY.MM.DD 04:00:00`) + [새로고침].
2. **P2 팔로워 달성 챌린지** — 잔디(1칸=하루, 농도=그날 발행 확정 건수 0·1·2·3·4+ 5단계).
3. **P3 업로드 릴레이** — Contributions Graph.
4. **다음 액션 인사이트** — 성과 기반 "다음에 만들 콘텐츠 방향" 제안 → C1로 연결("보기→행동").

🔗 **데이터**: `channel_insights_daily`(계정), `post_insights`(게시물), `challenge_logs`(일별 발행 건수, 예약은 실제 발행일에 적립).
🟦 **빈/예외**: 발행 이력 0 → "첫 발행 후 데이터가 쌓여요" / 팔로워 100명 미만 → 유입·이탈 분리 지표 **미제공 안내**(Graph API 제약).

## 3. 보조 플로우

- **콘텐츠 관리 `M1/M2`** — 칸반 6단계(기획중→기획완료→제작중→제작완료→예약업로드→업로드완료) 드래그/상태 표시. `M2` 캘린더↔리스트 토글, 관련 게시물·임시저장 토글. 🔗 `decks.status`.
- **자동화 `G1`** — 리드마그넷 규칙: 키워드 트리거 + 보낼 자료(정리본 링크) 등록, on/off. `GATE-한도`: 요금제 DM 한도(베이직 100·프로 1,000·프리미엄 ∞) 도달 시 안내. 🔗 `lead_magnets`, `dm_logs`.
- **계정 `A1→A2→A3`** — A1 프로필 모달(이름·이메일·구독정보·로그아웃) → A2 마이페이지(계정정보/구독·결제/데이터 전체·기간 삭제) → A3 요금제·결제(3종 비교, 월/연 토글 연 30%, 베타 무료). 🔗 `profiles`, `subscriptions`.

## 4. 비회원 플로우 *(IA "비회원으로 이용")*

```
L0 → [비회원으로 이용] → W0(비회원 셸, 제한) → 저장/발행 시도 → L2 가입 유도
```
- 체험 범위: 설문·컨셉·초안 생성까지 **둘러보기 가능**(가정), **저장·발행·연동은 가입 후**.
- ⚠️ 정확한 허용 범위는 **결정 대기**(§6).

## 6. 결정 대기

- **비회원 체험 허용 범위**(어디까지 둘러보기 vs 가입 게이트) — SPEC §6와 동일.
- **온보딩 내 연동 순서**: 컨셉 확정(O2) 후 연동(O3)이 강제인지, "나중에 연동" 스킵 허용인지.
- **검수 모달(C4) 위치**: 전용 풀스크린 vs 발행 버튼 옆 인라인 패널 — 와이어프레임에서 결정.
- **칸반 ↔ 캘린더 기본 뷰**: 진입 시 디폴트.

## 7. 와이어프레임 정합 메모

- 본 문서의 **화면 ID = 와이어프레임 프레임 1:1 라벨**로 쓰면 대조가 쉬움.
- 핵심 검증 포인트: ① 검수가 **독립 화면이 아니라 발행 내장 게이트(C4)**로 그려졌는지 ② 미연동 게이트(`GATE-연동`) 진입점 ③ 칸반 6열 ④ 인사이트 04:00 갱신 표기.
- 와이어프레임이 더 나은 동선을 제시하면 이 문서를 그쪽으로 갱신(역방향 OK). 충돌 시 우선순위 = **SPEC > IA > 본 문서**.
