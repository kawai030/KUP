
## Epic 0 — 생성 품질 기반 🔴 (벤치 선행)

### 0.1 단계별 프롬프트 상세 확정
- **무엇**: [파이프라인 §2](tech/Kup_생성파이프라인_설계.md) 초안을 *실제 운영 프롬프트*로 확정. 시스템(고정·캐싱) + 유저(①주제 ②전략 ③카피 ④자가점검) 4단계. 변수 주입 지점·JSON 스키마·금지표현/규제표현(금융·의료) 룰 명문화.
- **산출물**: `poc/llm-bench/prompts/` (단계별 .txt/.ts) + 변수 계약.
- **선행**: 없음 (지금 가능).
- **근거**: 파이프라인 §2, SPEC 3.3 검수 기준.

### 0.2 레퍼런스 카드뉴스 — 카피 모범 예시(few-shot)
- **무엇**: 잘 된 카드뉴스의 **텍스트 구조** 3~5개 큐레이션(cover 후킹→body 흐름→outro CTA→caption→해시태그). 이미지 아닌 텍스트만. few-shot으로 프롬프트에 주입.
- **산출물**: `poc/llm-bench/exemplars/*.json` (deck 스키마 형식).
- **선행**: 0.1.
- **근거**: 무수정 승인율 60~85%(SPEC 가드레일)을 직접 끌어올리는 레버. ⚠️ 스크랩 이미지를 자산화 ❌(저작권) — 분석/영감까지만.

### 0.3 카드뉴스 디자인 스펙 md
- **무엇**: 타이포 스케일·컬러 토큰·레이아웃 그리드·슬라이드 아키타입(cover/body/outro 변형)을 문서화. 레퍼런스 ~10개는 이 스펙 도출의 *참고*.
- **산출물**: `docs/tech/Kup_카드뉴스_디자인스펙.md`.
- **선행**: 없음 (0.2와 병렬).
- **근거**: 일관된 비주얼의 SoT. 렌더 템플릿(0.4)이 이걸 구현.

### 0.4 렌더 템플릿 2~3종 제작 (자체)
- **무엇**: 디자인 스펙(0.3) 기반 자체 템플릿. 현행 `poc/render-deck.js`(sharp+SVG) 확장 vs Playwright(HTML/CSS) 결정 포함. 브랜드컬러 주입·글자수 제약([파이프라인 §1.3](tech/Kup_생성파이프라인_설계.md)) 검증.
- **산출물**: 템플릿 N종 + 샘플 PNG.
- **선행**: 0.3.
- **근거**: 개발계획 §7 렌더 결정.

### 0.5 골든셋 + 채점 루브릭 확정
- **무엇**: 벤치 고정 시나리오 3종(재테크/디저트 고위험/자유주제) 입력 + 기대 출력 기준 + [개발계획 §4 루브릭](tech/Kup_기술스택_개발계획.md) 채점 시트.
- **산출물**: `poc/llm-bench/scenarios/*.json` + 채점 템플릿.
- **선행**: 0.1, 0.2.
- **근거**: 벤치 공정성(같은 입력·스키마·max_tokens).
